3. The two boundaries that carry the safety posture

TB-1 — Admission (write path)
Untrusted conversation becomes stored state only after extraction (typed, resolved) and a hard PII gate. There is deliberately no post-storage redaction path: by the time data is on the output it is already in the index and backups (P6). The gate is the only correct place.
TB-2 — Isolation (read path)
Stated as "no query path can express a cross-tenant read," not "every path filters" (C8). Two enforcement layers (RLS + constructor-scoped repository) mean an application bug is caught by the database and an RLS misconfig is caught by the code shape. D3 measured why this must be structural: the naive filter-mindset leaked on 7/11 queries, and better similarity makes near-duplicate foreign memories more retrievable, not less.
