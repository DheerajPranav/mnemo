### Quiz-me gate (3 questions logged; answered by implementer, surfaced to user at handoff)
1. *Design decision:* Why is tenant a constructor argument rather than a method parameter?
   → So no method can *express* a cross-tenant read; the wrong tenant cannot be passed because there is nowhere to pass it, and a static check can mechanically enforce it (test_isolation §1). RLS (ADR-005) is defence-in-depth behind this, not the primary guarantee.
2. *Edge case:* Two tenants both have an account literally named "Acme" (m101/m102 are T2's Acme). What stops T1 seeing them?
   → Every read is `WHERE tenant_id = self._tenant_id`; account is only an *in-tenant* narrowing. Probe A2 confirms current_facts(account="Acme") returns 0 T2 rows.
3. *Change impact:* If a future PR adds `def get(self, id, tenant=None)`, what happens?
   → test_isolation.test_no_public_method_accepts_a_tenant_parameter fails the build (I1 violated). The DoD gate "no context-graph invariant violated" blocks the merge.
