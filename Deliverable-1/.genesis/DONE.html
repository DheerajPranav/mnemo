1 · The Locked Spec (what we are building)

Cognitive job: Maintain a durable, current, tenant-isolated, PII-safe memory of what is true about each account over time, and select the currently-valid subset into a bounded context window on demand — so a GTM assistant answers from what is still true, never from a superseded, foreign-tenant, or sensitive fact. (Mnemo, per the D4 design in design/system_design.pdf.)

Inputs: conversation turns / notes carrying (tenant_id, actor, timestamp); a retrieval query with a token budget  →  Outputs: typed, validity-stamped facts persisted with provenance; a ranked, tenant-isolated, currently-valid, budget-bounded context injection or an explicit abstention — every decision traced.

Autonomy level: autonomous on the read/write path (extraction · admission · retrieval · ranking · injection); high-risk lifecycle actions (bulk deletion, consolidation merges) are logged and reversible.  |  Human-in-the-loop points: the L4 VERIFY quiz-me gate is the human checkpoint each milestone; deletion / consolidation merges are reviewable.

Failure tolerance: zero for the unrecoverable failures — F10 cross-tenant leak, F11 PII retention, F9 deletion evasion; for the recoverable ones, graceful degradation (abstain) is preferred over a confident wrong answer.  |  Trust boundary: TB-1 admission (untrusted conversation text → typed facts; the store is an indirect prompt-injection channel, threat R4) and TB-2 tenant isolation (row-level, enforced in the repository constructor).

Filled during Genesis step G0 (Cognitive Design). Traces to reconstruction/first_principles.md (C1–C10), experiments/ (measured F1–F11), and design/threat_model.md (TB-1/TB-2, R4).

Dependency direction inward, zero cycles (modular-architecture gate)

python3 -m py_compile implementation/mnemo/*.py clean AND python3 -m unittest discover -s implementation/tests green
One verify pass by a separate agent/model (L4)
No context-graph.json invariant (I1–I6) violated by the change

A milestone is DONE only when every box above is checked by the verifier. Pull each gate from the relevant agentic-swe-master phase gate.

Per-milestone binary gates — the D4 sprint gates G0–G2, computed by a runnable script (never narrated). These are the checker's oracle. Numbers-to-beat come from the D3 baseline (experiments/baseline_results.csv).

G0 (M1, blocking): the cross-tenant adversarial suite returns 0 foreign-tenant rows across all 11 D3 queries — gate_g0_isolation.py exits 0. Invariant I1. Closes F10.
G1 (M2): an admit carrying a phone / medical token stores nothing for that candidate — PII exposures = 0 (baseline: 3) — gate_g1_pii.py exits 0. Invariant I2. Closes F11.
G2 (M3): on the D3 fixed set, supersession-failure < 0.80 (baseline) AND cross-tenant leak = 0 (baseline: 7/11) — gate_g2_baseline.py exits 0. Invariants I3+I4. The design's core claim, proven or falsified.
